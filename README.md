# ERP Coffee V3

Учебный мини-проект ERP для кофейни на `React + Vite` в простом и расширяемом формате.

## Что реализовано

- **Страница товаров**:
  - карточки напитков;
  - кнопка `Приготовить` (сохранение заказа + списание ингредиентов);
  - кнопка `Рецепт` (модальное окно с расходом ингредиентов);
  - блок `Управление товарами` с кнопкой добавления нового товара через модалку.
- **Правая панель (как в V1)**:
  - текущие остатки ингредиентов;
  - предупреждения по порогам;
  - последние заказы.
- **Страница ингредиентов**:
  - карточки ингредиентов с остатками и порогом;
  - кнопка `Изменить порог` на карточке (через модалку);
  - правый блок `Управление ингредиентами` с кнопками:
    - `Добавить ингредиент` (модалка);
    - `Открыть закупку` (модалка пополнения остатков).
- **Валидация форм**:
  - предупреждения, если обязательные поля не заполнены или числа некорректны.
- **Локальные изображения**:
  - используются файлы из `public/images`, без внешних URL.
- **Сохранение состояния**:
  - данные (`menu`, `ingredients`, `orders`) сохраняются в `localStorage`.

## Технологии

- `React`
- `Vite`
- чистый `CSS` (без UI-библиотек)

## Запуск проекта

1. Установите зависимости:

```bash
npm install
```

2. Запустите dev-сервер:

```bash
npm run dev
```

3. Откройте адрес из консоли (обычно):

`http://localhost:5173`

## Быстрая проверка сценариев

1. На странице товаров нажмите `Приготовить`:
   - заказ должен появиться в `Последние заказы`;
   - остатки в правом блоке должны уменьшиться.
2. Нажмите `Рецепт` на карточке товара:
   - откроется модалка с составом и расходом.
3. Добавьте новый товар через `Управление товарами`:
   - заполните название, цену и рецепт;
   - проверьте, что товар появился в карточках.
4. Перейдите на страницу ингредиентов:
   - добавьте ингредиент через модалку;
   - выполните закупку через модалку;
   - измените порог на карточке ингредиента.
5. Перезагрузите страницу:
   - изменения должны сохраниться.
